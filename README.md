# Ntek Printing - Business Website

A modern, full-stack business website for Ntek Printing services built with React, Express, and TypeScript. Features a contact form system with admin dashboard and Supabase integration.

## 🌟 Features

### 📱 Client-Side

- Modern, responsive UI built with React 18 and TailwindCSS
- Component library using Radix UI primitives
- Client-side routing with Wouter
- Form handling with React Hook Form
- Toast notifications for user feedback
- Type-safe API requests with TypeScript
- Optimized performance with Vite build tool

### 🖥️ Admin Dashboard

- Real-time view of all contact form submissions
- Status tracking (New, Contacted, Closed)
- Sortable by date, status, and other fields
- Visual status indicators with color coding
- Responsive table layout for all devices

### 🔧 Server-Side

- Express.js backend with TypeScript
- Supabase integration for data storage
- RESTful API endpoints
- Environment variable configuration
- Error handling and logging
- Production-ready setup

## 🚀 Quick Start

### Prerequisites

- Node.js (v20 or higher)
- npm (Node Package Manager)
- A Supabase account and project

### Environment Setup

1. Clone the repository:

   ```bash
   git clone <repository-url>
   cd ntek-printing
   ```

2. Install dependencies:

   ```bash
   npm install
   ```

3. Create a `.env` file in the root directory:
   ```env
   NODE_ENV=development
   SUPABASE_URL=your_supabase_project_url
   SUPABASE_ANON_KEY=your_supabase_anon_key
   ```

### Database Setup

1. Go to your Supabase project
2. Open the SQL editor
3. Run the following SQL to set up the contact submissions table:

   ```sql
   -- Create the table
   create table if not exists contact_submissions (
       id bigint generated by default as identity primary key,
       created_at timestamp with time zone default timezone('utc'::text, now()) not null,
       name text not null,
       email text not null,
       phone text,
       service text,
       message text not null,
       status text default 'new'::text check (status in ('new', 'contacted', 'closed'))
   );

   -- Enable RLS
   alter table contact_submissions enable row level security;

   -- Create policies
   create policy "Enable insert for anonymous users"
       on contact_submissions
       for insert
       to anon
       with check (true);

   create policy "Enable read for anonymous users"
       on contact_submissions
       for select
       to anon
       using (true);

   -- Grant permissions
   grant usage on sequence contact_submissions_id_seq to anon;
   grant all on contact_submissions to anon;
   ```

### Running the Application

#### Development

```bash
npm run dev
```

The application will be available at http://localhost:4000

#### Production Build

```bash
npm run build
npm start
```
